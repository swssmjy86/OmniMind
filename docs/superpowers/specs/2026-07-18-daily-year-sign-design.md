# 오늘의 일진 — 년도·띠 기반 무료 훅 설계

> 작성 2026-07-18. P9 1단계(밤의 따뜻함) 후속. 사용자 결정 3건 반영:
> ① 홈 상단 일진 카드 제거, 밤의 서재로 통일 ② 프로필 있으면 /daily에서 자동 심화
> ③ 깊이 = 공통 일진 + 띠 관계 + 년간 십성.

## 1. 목적

온보딩("나를 알아보기") 없이도 **출생 년도 하나만으로** 오늘의 일진을 자세히 보여주는
무료 훅. 밤의 서재의 '오늘의 일진' 카드(달지기)가 진입점이다.

- 흐름: 서재 카드 클릭 → `/daily` → 년도 입력 → **"1990년생, 말띠시군요"** → 상세 일진
- 프로필 있는 로그인 사용자는 년도 입력 없이 기존 심화 일진을 그대로 본다.

## 2. 화면 (`/daily`, 페르소나: 달지기)

`force-dynamic` 서버 컴포넌트. 세 가지 상태:

| 상태 | 조건 | 내용 |
|---|---|---|
| **심화** | 로그인 + 프로필 | 홈에 있던 일진 섹션 그대로 이사: 일간 십성 개인화 + LLM 문단 + 하루기록(DailyRecorder) + 공유 카드(ShareSheet) + 지난 이야기 링크 + 근거 접이식 |
| **띠 일진** | `?year=YYYY` 유효(1900~올해) | ① "OOO띠시군요" 헤더(년간지 병기) ② 공통 일진(기운·마음가짐·색·키워드·행운 — 기존 `assembleDaily`) ③ 띠 단락: 년지 × 오늘 일진 지지의 전통 관계 ④ 년간 십성 문단("타고난 해의 기운으로 보면" 라벨) ⑤ 입춘 고지 한 줄 ⑥ CTA "생년월일시까지 알려주시면 더 깊이 → 나를 알아보기" |
| **입력 폼** | 그 외 | 달지기 인사 + 4자리 년도 입력(GET 제출). 잘못된 년도는 부드러운 안내와 함께 폼 재표시 |

- **구현 방식(확정): URL 파라미터 + 서버 렌더.** 엔진(절기 테이블 포함)을 클라이언트
  번들에 싣지 않는다. 링크 공유 가능.
- 입력 년도는 localStorage에 기억(클라이언트 폼 컴포넌트가 prefill) — 자동 리다이렉트는
  하지 않는다(YAGNI, 잘못 저장 시 빠져나올 수 없는 루프 방지).

## 3. 홈 변경

- 상단 일진 카드 **제거** (동행일·마일스톤 배지 헤더는 유지).
- `ProductShelf`의 daily 제외 필터 제거 → '오늘의 일진' 카드(누구나 무료)가 셸프 맨 위.
- `products.ts`: daily `href: "/" → "/daily"`.
- DailyRecorder·ShareSheet·LLM 문단·근거 접이식은 `/daily` 심화 상태로 이동.

## 4. 계산 (순수 함수, LLM 없음)

`src/lib/engine/year-sign.ts` **신규** — 네트워크·IO 없음, 기존 엔진 원칙 준수.

```ts
ZODIAC_ANIMALS: ["쥐","소","호랑이","토끼","용","뱀","말","양","원숭이","닭","개","돼지"]
  // EARTHLY_BRANCHES(자~해)와 같은 인덱스
yearSign(year: number): { stem; branch; ganzhi; animal }   // (year-4)%10, (year-4)%12
branchRelation(a: number, b: number):
  "육합" | "삼합" | "충" | "형" | "해" | "파" | null
```

- **관계 테이블**: 육합 6쌍 / 삼합 4국(같은 국의 서로 다른 지지) / 충 6쌍(대칭 6칸) /
  형(인사신·축술미 상호, 자↔묘, 진·오·유·해 자형=같은 지지) / 해 6쌍 / 파 6쌍.
- **중복 판정 우선순위(확정)**: 충 > 육합 > 삼합 > 형 > 해 > 파 > 무관계.
  (인-해, 사-신처럼 겹치는 쌍은 이 순서로 하나만 보여준다 — 사용자에게 단일 판정.)
- **띠는 양력 연도 기준.** 입춘 경계는 년도만으로 판별 불가 → 화면에 정직 고지:
  "띠는 양력 연도 기준이에요. 입춘(2월 초) 무렵에 태어났다면 앞뒤 띠일 수 있어요."
- 년간 십성 문단은 **기존 `computeDaily(date, element, stem)` 재사용** — 년간을 넘긴다.
  새 해석 엔진 없음.

## 5. 해석 문구

`src/lib/interpret/content/year-sign.ts` 신규 — 관계별 한 단락(달지기 톤, ~요체):

| 관계 | 방향 |
|---|---|
| 육합 | 아주 잘 통하는 날 |
| 삼합 | 흐름이 모이는 날 |
| 충 | 들뜨기 쉬운 날, 중심 잡기 |
| 형 | 마찰이 있을 수 있는 날, 한 템포 쉬어가기 |
| 해 | 어긋나기 쉬운 날, 서두르지 않기 |
| 파 | 계획이 틀어질 수 있는 날, 여유 두기 |
| 무관계 | 평온하게 흐르는 날 |

전 문구 톤 가드(`하세요`/`~니다`/공포 문구 금지) 테스트 필수. "나쁜 날" 단정 금지 —
충·형도 마음가짐 제안으로 끝낸다(§5.4 지적인 따뜻함).

## 6. 해석 축 위계(§3) 정합

띠 관계·년간 십성 모두 **팔자 주축**의 재료다. 보조축(MBTI·혈액형) 없음 — 위계 충돌 없음.
프로필 심화 상태는 기존 일간 기반이 그대로 우선한다(년도 기반보다 깊은 축).

## 8. 테스트

- `yearSign`: 1990→경오·말, 2000→경진·용, 1900→경자·쥐, 2026→병오·말 (음수 모듈로 안전)
- `branchRelation`: 자축=육합, 신자진=삼합, 자오=충, 인사=형(해보다 우선 아님 — 우선순위에
  따라 형), 인해=육합(파보다 우선), 사신=육합, 자유=파, 자미=해, 진진=형(자형), 자진=삼합
- 문구: 관계 7종 전부 존재·톤 가드 통과
- YearForm: 렌더·유효성(min/max)·localStorage prefill
- DailySignSection: 띠 헤더·4개 블록·CTA·입춘 고지 렌더
- ProductShelf: 5장(일진 포함, 일진이 첫 번째), 일진 href=/daily
- 홈: 일진 섹션 부재 확인은 스모크(async 페이지라 단위 렌더 제외 — 기존 원칙)

## 9. 비목표

- 띠별 운세 콘텐츠 DB·LLM 개인화(무료 훅은 규칙 기반으로 충분)
- 년도 기반 결과의 서버 저장(비로그인 — 저장할 곳 없음, localStorage만)
- 입춘 경계 정밀 판정(월일 입력 없이는 불가 — 정직 고지로 대체)
