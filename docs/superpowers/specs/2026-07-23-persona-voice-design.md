# 페르소나 전면 몰입 — 말투·특징의 템플릿 반영 설계

> 작성 2026-07-23. `docs/persona-plan.md`(7인 체제 정의서)의 후속.
> 결정: **전면 몰입** — 항상 표시되는 템플릿 본문(2단)까지 담당 페르소나의 말투로 쓴다.
> 범위: **7종 전부** (총운·오늘의운세는 점검 결과 기존 카피가 이미 서온·달지기 결이라 유지).

## 배경 — 무엇이 어긋나 있었나

| 계층 | 개편 전 |
|---|---|
| 상품 페이지 인사(`persona.greeting`) | ✅ 페르소나 반영 |
| LLM 개인화 문단(3단, `toneInstruction`) | ✅ 페르소나 반영 |
| 템플릿 본문(2단 — ①~④ 섹션, 항상 동작) | ❌ 전 상품 중립 요체 |

연애운은 홍연(반말) 상품인데 본문이 "~해요", 재물운은 금오(하오체) 상품인데 본문이
"~해요"로 나가고 있었다. LLM 쿼터가 없을 때(템플릿만 동작)는 페르소나가 전혀 드러나지
않는 구조.

## 설계

### 1. Voice — 어미 문법 4갈래 (`personas.ts`)

```ts
export type Voice = "yo" | "banmal" | "hao" | "jiyo";
```

- 페르소나마다 `voice` 필드: 달지기·서온·벼리·연리=`yo`, 홍연=`banmal`, 금오=`hao`, 온새=`jiyo`
- **어미와 결의 분리:** 공용 문구(신강약·오행·대운)는 어미 4갈래만 알고, 캐릭터의 결
  (비유·리듬·호칭)은 상품별 전용 문구가 담는다. 요체 4인(달지기·서온·벼리·연리)의 차이는
  전용 문구의 리듬이 만든다 — 벼리는 짧은 문장, 연리는 나무 비유.

### 2. 공용 문구의 말투 변형

호출부 호환을 위해 전부 `voice: Voice = "yo"` 선택 인자. 내용·불변 표기("X의 기운이 N개",
"나란히", 계절 이름, "대운")는 네 갈래 동일, 어미·호칭(당신/너/그대)만 다르다.

- `strength.ts` — `strengthText`, `patternText`, `patternsText`
- `elements.ts` — `ELEMENT_BALANCE_TEXT` (성향·계절·골격·마무리 전부)
- `daeun.ts` — `DAEUN_SEASON_TEXT`, `daeunSeasonText`
- `chongun.ts` — `daeunSeasonBody` (세 분기 골격)

### 3. 상품별 본문 재작성 (`credit-readings.ts`, `match.ts`)

| 상품 | 페르소나 | 반영 |
|---|---|---|
| 직업운 | 벼리 | 짧고 단단한 요체, 쇠·연장·담금질 결. 직업 예시 "거기서 결이 서요" |
| 연애운 | 홍연 | 다정한 반말 + 부름말(`새벽아,` — 받침 따라 아/야) |
| 재물운 | 금오 | 호쾌한 하오체(~이오/~소), 호칭 "그대", 물길 비유 |
| 결혼운 | 온새 | 포근한 지요체, 느린 리듬, 나무·길 비유 |
| 궁합 | 연리 | 그윽한 요체(~지요 여운), 나무·바람·그늘 비유, 양쪽 공평 |
| 총운 | 서온 | 기존 카피가 이미 차분한 요체(사서의 결) — 유지 |
| 오늘의운세 | 달지기 | 기존 카피가 이미 속삭이는 요체(문지기의 결) — 유지 |

- 상품→voice는 `PRODUCT_PERSONA`(파생)에서 다시 파생 — 매핑을 옮겨 적지 않는다.
  총운도 `PERSONAS[PRODUCT_PERSONA.chongun].voice`로 파생(SSOT 일원화, 리뷰 반영).
- 부름말(리뷰 반영): 홍연 반말 "새벽아/하나야", 금오 하오체 "새벽이여/하나여"(연극적
  하오체 — "님+그대" 존대 혼종 제거), 그 외 "새벽님". 이름이 비었거나 한글로 끝나지
  않으면 어미 없이 안전하게 줄인다.
- 호칭이 담긴 섹션 제목은 전부 말투를 따른다(리뷰 반영): 보조축 "너에게/그대에게/
  당신에게 드러나는 방식" + LLM 문단 제목 "너만을/그대만을/당신만을 위한 이야기".
- LLM 문단(⑤)은 기존처럼 `toneInstruction`이 말투를 정하고, 프롬프트에 "위 섹션과 같은
  어조 유지" 한 줄을 더해 2단·3단의 목소리가 이어지게 한다.
- **내용 불변 원칙:** 공용 문구의 말투 변형은 어미·호칭만 다르고 의미·수식은 동일해야
  한다(같은 사주가 상품마다 다른 성격 주장을 하면 안 된다 — 리뷰에서 드리프트 3건 정정).

### 4. 캐시 재생성 (`hash.ts`)

`READING_TEMPLATE_VERSION = 2`를 **문구가 바뀐 상품의 호출부만** 해시에 섞는다(`tv`,
크레딧 4종·궁합). 기본값 1은 필드를 아예 넣지 않아 v1 시절 해시와 동일 — 문구가 안
바뀐 총운의 기존 캐시와 거기 붙은 후기(reading_reviews)가 그대로 살아 있다(리뷰 반영:
전역 무효화는 후기 고아·중복 집계를 만든다). 문구가 바뀐 상품의 옛 행·후기는 재생성 후
고아로 남는다 — 정식 오픈 전이라 수용하고, 오픈 후 개편 시에는 후기 이관을 함께 설계한다.

### 5. 톤 가드 제약 (전 말투 공통)

- `~니다` 금지 → 온새의 "~답니다"는 쓰지 않는다(정의서의 말투 샘플 중 지요/그래요만).
- `~하세요` 금지, 낙인형 "사람이에요/야/죠" 금지 — 전 변형이 기존 tone-guard 테스트를 통과.

### 6. 테스트

- 말투 변형 전수 × 톤 가드: strength(3×4)·elements(4)·daeun 분기(3×4).
- 말투 서명: 연애=`요[.!?]` 부재+부름말 / 재물=`[오소][.!?]` 존재+"그대" / 결혼=`지요.` /
  직업=요체+님 호칭.
- 기존 불변 표기 유지: "기운이 N개"·"나란히"·"대운"·`합(合)`·온도 `°`.
