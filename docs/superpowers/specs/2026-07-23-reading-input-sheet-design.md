# 풀이 진입 입력 시트 — 생년월일·시간·MBTI·혈액형 설계

> 작성 2026-07-23. 대상: 총운·직업운·연애운·재물운·결혼운 5종.
> 결정(사용자 확인): ① MBTI·혈액형은 **보조축으로 풀이 본문에 반영**(2026-07-20 제거분의
> 부분 복원) ② **비로그인도 시트 입력만으로 풀이를 바로 본다**(2026-07-21 "비로그인은
> 오늘의운세만" 결정을 뒤집음 — `GUEST_READING_ACCESS=true` 재개방).

## 요구

풀이 5종의 "풀이 보러 가기"를 누르는 순간, 필요한 입력(생년월일·태어난 시·MBTI·혈액형)이
없으면 **입력 창(바텀시트)** 이 나타난다. 입력하면 그 자리에서 풀이가 보인다.

## 흐름

카드 클릭 → 상품 페이지 진입(라우팅은 그대로) → 페이지가 입력 상태를 보고 시트를 띄운다.
딥링크로 직접 들어와도 같은 시트가 뜬다(우회 없음).

| 사용자 상태 | 동작 |
|---|---|
| 비로그인, draft 없음/불완전 | 시트 **전체 모드**(생년월일·시·성별·MBTI·혈액형) → localStorage draft 저장 → 즉시 계산(게스트 경로, LLM 없음) |
| 비로그인, draft 있으나 MBTI·혈액형 없음 | 시트가 **빠진 것만**(MBTI·혈액형) → draft 갱신 → 계산 |
| 로그인, 프로필 없음 | 기존 온보딩 CTA 유지(온보딩이 이미 그 입력 화면) |
| 로그인+프로필, MBTI·혈액형 없음 | 시트 **특성 모드**(MBTI·혈액형만) → `profiles.mbti`/`blood_type` 저장(서버 액션) → 새로고침 후 풀이 |
| 입력 완비 | 시트 없이 바로 풀이 |

- 필수: 생년월일(전체 모드), MBTI, 혈액형. 태어난 시는 "모름" 허용(엔진 `timeUnknown`),
  성별은 선택(대운용) — 오늘의운세 시트와 같은 결.
- 닉네임은 받지 않는다. 기존 draft/프로필 것을 쓰고, 없으면 호칭 생략(방어 처리 완료).

## MBTI·혈액형 보조축 (부분 복원)

- **엔진에 되돌리지 않는다.** 계산이 없는 룩업이라 순수 계산 계층·`PROFILE_CONTEXT_VERSION`
  승격(전 프로필 재계산)을 피한다. 해석 계층 신설 모듈 `interpret/content/traits.ts`가
  2026-07-20 제거분의 문구(MBTI 4축·혈액형 4유형)를 계승한다.
- `assembleCreditReading`/`assembleChongun`이 선택 인자 `traits {mbti, blood}`를 받아:
  - 크레딧 4종: 보조축 섹션("~에게 드러나는 방식") 끝에 보조 문장 추가
  - 총운: 마지막에 "겉과 속" 섹션 추가
  - 사주가 주어·MBTI/혈액형은 수식(해석 축 위계) · 페르소나 말투 4갈래 준수 · 없으면 기존 그대로(폴백)
- 저장처: 게스트 `Draft.mbti/blood`(신규 optional 필드), 로그인 `profiles.mbti/blood_type`
  (0014에서 nullable로 이미 존재 — 컬럼 재사용, 마이그레이션 불필요).

## 캐시

로그인 풀이 해시에 traits를 **있을 때만** 포함한다(`{ctx, traits}` 래핑). 없으면 기존
해시 그대로 — traits를 아직 안 넣은 사용자의 총운 캐시·후기가 살아 있고, 입력하는 순간
자연 재생성된다.

## 범위 밖

- 궁합: 자체 상대 입력 폼 유지(시트 없음). 단 `GUEST_READING_ACCESS` 재개방으로 게스트
  열람은 함께 열린다(기존 GuestMatchDeepGate 경로).
- 오늘의운세(자체 팝업), 온보딩 화면 개편, LLM 프롬프트에 MBTI·혈액형 별도 지시(섹션
  본문에 이미 실려 자연 전달).

## 테스트

- traits 문구: 16 MBTI × 4말투 × 혈액형 4종 톤 가드 전수, 부분 입력(한쪽만) 폴백
- 조립: traits 있음/없음 — 섹션 수·보조 문장 유무, 페르소나 말투 서명 유지
- draft: 신규 필드 저장/로드/하위 호환(구버전 draft에 필드 없음)
- 시트: 전체/특성 모드 렌더, 필수값 없이 제출 불가, MBTI 4축 조합
- 해시: traits 유무에 따른 캐시 키 분기(없으면 기존과 동일)
