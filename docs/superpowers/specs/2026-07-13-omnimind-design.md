- 상태: 승인됨

## 1. 개요

**OmniMind(옴니마인드)** 는 사주팔자(만세력)·혈액형(4유형)·MBTI(16유형)·별자리(12궁)를 종합해 사용자가 '온전한 나'를 발견하도록 돕는 **모바일 웹 기반 서비스**다. 이름은 Omni(모든 것) + Mind(마음)의 결합.

**핵심 가치 제안:** 단순 운세가 아닌 "생애 주기별 멘탈 가이드" — 인생의 고비마다 성향과 운의 흐름을 종합해 의사결정을 돕는 **동반자**.

### 확정된 요구사항

| 항목 | 결정 |
|------|------|
| 타깃 사용자 | 20대 후반~30대 여성 |
| 플랫폼 | 모바일 웹 우선, 데스크톱 반응형 |
| 해석 엔진 | 하이브리드 — 규칙 기반 계산 + LLM 감성 해석 |
| 계정/데이터 | 소셜 로그인(카카오·구글) + 서버 저장 |
| 수익화 | 광고 기반 무료 시작 → 가입자 성장 후 프리미엄 구독 전환 |
| 운영 비용 | **월 고정비 0원** — 무료 티어·무료 API만으로 전체 구성 |
| 출시 전략 | 4대 기능 전부 목표, 단 작게 나눠 빠르게 출시하고 꾸준히 개선 |
| 개발 | 1인 + Claude Code 주도 |
| 성공 기준 | 완성도 — 4대 기능이 모두 안정적으로 동작하는 서비스 |

### 4대 핵심 기능

1. **나에 대한 이해** — 4개 분석 체계를 종합한 '온전한 나' 프로필
2. **그날의 기운/성향** — 데일리 가이드 (매일 갱신)
3. **의사결정 도우미** — 인생의 고비에서 프로필+운기 종합 조언
4. **대화형 AI 동반자** — 프로필 맥락을 기억하는 공감형 챗

## 2. 아키텍처 (무료 우선)

```
[모바일 웹 브라우저]
        │
        ▼
┌──────────────────────────────────┐
│  Next.js — Vercel 무료 티어 배포   │
│  ├─ 프론트: 모바일 우선 UI         │
│  └─ API Routes                    │
│      ├─ 계산 엔진 (순수 TS)        │
│      ├─ 템플릿 문장 엔진           │
│      └─ LLM 해석 (무료 티어)      │
└──────┬───────────┬───────────────┘
       │           │
       ▼           ▼
 [Supabase]   [Gemini API 무료 티어]
  카카오/구글    쿼터 초과 시
  로그인 + DB    템플릿 엔진으로 자동 대체
```

### 인프라 구성 (전부 무료)

| 구성 요소 | 서비스 | 무료 범위 |
|-----------|--------|-----------|
| 호스팅/배포 | Vercel Hobby | 개인 프로젝트 충분 |
| DB + 소셜 로그인 | Supabase Free | DB 500MB, 월 활성 사용자 5만 |
| LLM | Google Gemini API 무료 티어 | Flash 모델 일일 무료 쿼터 |
| 광고 | 카카오 AdFit / Google AdSense | 수익원 (비용 아님) |

### 핵심 원칙: 계산과 해석의 분리

- **계산 (규칙 기반, 0원):** 만세력(사주팔자·오행·십성), 별자리 산출, MBTI/혈액형 분류. 결정론적 공식이므로 100% 코드로 구현. LLM에 맡기지 않는다 (오류 위험 제거).
- **해석 (LLM/템플릿):** 계산 결과를 받아 "따뜻한 문장"으로 풀어내는 역할만.

### 3단 해석 엔진 — 무료를 버티는 구조

```
1단: 계산 엔진 (규칙 기반)              ← 항상 동작, 0원
2단: 템플릿 문장 엔진 (사전 작성 문구 조합) ← 항상 동작, 0원, 기본 품질 보장
3단: LLM 개인화 레이어 (Gemini 무료 쿼터) ← 쿼터 있을 때만, 소진 시 2단으로 대체
```

- 템플릿 DB는 유형 조합(일간×오행 분포×MBTI×혈액형×별자리) 단위 해석 문구를 개발 단계에서 사전 작성 — 1회성 작업, 운영 비용 없음.
- LLM 없이도 서비스가 **완전히** 동작한다. 쿼터 소진은 장애가 아니라 정상 경로.

### 해석 Provider 교체 구조 (수익 발생 시 유료 전환 대비)

```
해석 요청 → [해석 Provider 인터페이스] ← 설정값 하나로 교체
                 ├─ 0단계: 템플릿 전용 (완전 무료)
                 ├─ 1단계: Gemini 무료 티어 (출시 시점) ← 시작
                 └─ 2단계: 유료 모델 — Claude / Gemini Pro (구독 수익 발생 후)
```

- 모든 LLM 호출을 단일 인터페이스 뒤에 배치. 유료 전환 시 어댑터 하나 추가 + 설정 변경으로 끝.
- 프리미엄 구독자 = 유료 모델, 무료 사용자 = 무료 티어의 이중 운영 가능.

### 비용 통제 장치

- **프로필:** 가입 시 1회 생성 후 영구 저장
- **데일리 가이드:** 사용자별 하루 1회 생성 후 캐시 (재방문 시 LLM 호출 0)
- **AI 챗:** 무료 하루 10회 제한 (쿼터 보호 + 프리미엄 판매 포인트)

## 3. 화면 구성 (모바일 우선)

```
[첫 방문]                         [재방문]
스플래시/랜딩 ─► 소셜 로그인 ─►      홈 (오늘의 가이드)
      "모든 나를 잇다, 옴니마인드"        │
              │                    하단 탭 네비게이션
              ▼                ┌─────┬─────┬─────┬─────┐
     온보딩 (대화하듯 단계별)     │ 홈  │ 나  │ 고민 │ 마음 │
     ① 생년월일시 입력           │오늘의│온전한│의사 │ AI  │
     ② 혈액형 선택              │가이드│ 나  │결정 │ 챗  │
     ③ MBTI 선택 (모르면 안내)   └─────┴─────┴─────┴─────┘
     ④ 별자리 자동 산출 확인
              │
              ▼
     "온전한 나" 프로필 첫 공개 ✨
```

- **온보딩은 설문지가 아니라 대화처럼** — 한 화면에 하나씩, 따뜻한 문구와 함께. 첫 3분이 승부처.
- **홈 = 데일리 가이드** — 매일 새 콘텐츠로 재방문 유도. 광고는 콘텐츠 흐름을 깨지 않는 위치(가이드 하단 등)에만.
- 4개 탭 = 4대 기능, 구조가 기획과 1:1 대응.
- 출생 시간 미상 사용자를 위한 "시간 몰라요" 옵션 제공 (시주 제외 해석).

## 4. 데이터 모델 (Supabase / PostgreSQL)

```
users            — 소셜 로그인 계정 (Supabase Auth 연동)
profiles         — 생년월일시, 혈액형, MBTI, 별자리 + 계산 결과(사주팔자, 오행 분포) JSON
interpretations  — 생성된 해석 캐시 (종류: 프로필/데일리/조언, 날짜, 본문, 생성 방식 0/1/2단계)
chat_messages    — AI 챗 대화 기록 (동반자의 '기억')
templates        — 해석 템플릿 문구 DB (유형 조합 키 → 문구)
usage_counters   — 사용자별 일일 LLM 사용량 (챗 횟수 제한용)
```

- 계산 결과는 profiles에 1회 저장 후 영구 재사용.
- 해석문은 interpretations에 캐시 — 같은 것을 두 번 생성하지 않음 (무료 쿼터 보호가 데이터 모델에 반영).

## 5. 브랜딩 설계

### 5.1 슬로건 운용 체계

| 슬로건 | 노출 위치 | 목적 |
|--------|-----------|------|
| "모든 나를 잇다, 옴니마인드" | 랜딩 히어로, 로그인 화면 | 첫인상 — 서비스 정의 |
| "당신의 마음을 가장 잘 아는 모든 데이터" | 온보딩 완료 → 프로필 공개 직전 로딩 | 기대감 최고조 순간에 신뢰 각인 |
| "나보다 나를 더 잘 아는, 옴니마인드" | 공유 카드 하단, SNS 광고 소재 | 외부 노출용 |

### 5.2 컬러 시스템 (디자인 토큰)

| 토큰 | 색상 | HEX | 용도 |
|------|------|-----|------|
| `--warm-base` | 웜 베이지 | `#F5EFE6` | 배경 |
| `--warm-surface` | 아이보리 | `#FDFBF7` | 카드/시트 표면 |
| `--accent-coral` | 차분한 코랄 | `#E8927C` | 주요 버튼, 강조 |
| `--primary-green` | 딥 그린 | `#2D5A4A` | 헤더, 로고, 네비 활성 |
| `--text-main` | 웜 차콜 | `#3E3A36` | 본문 (순검정 금지) |
| `--text-soft` | 모카 그레이 | `#8A8178` | 보조 텍스트 |

규칙: 블루 계열 전면 금지. 경고/오류도 코랄 톤. 다크 모드는 딥 그린 바탕 + 베이지 텍스트.

### 5.3 타이포그래피 & 무드

- 서체: **Pretendard**(본문) + 프로필·데일리 제목에만 명조 계열(RIDIBatang 또는 Noto Serif KR) — "편지를 받는 느낌"
- 모서리: 큰 라운드(16~24px), 옅고 부드러운 그림자. 딱딱한 직각·강한 그림자 금지
- 아이콘/일러스트: 가는 라인 스타일, 달·별·잎사귀 모티프

### 5.4 문체 규칙 — '지적인 따뜻함' (모든 카피·해석문·AI 응답에 강제)

| 규칙 | ❌ 금지 | ✅ 권장 |
|------|---------|---------|
| 분석 용어 금지 | "분석 결과입니다" | "당신의 이야기를 읽어봤어요" |
| 단정 금지 | "당신은 ~한 사람입니다" | "~한 면이 있으시군요" |
| 명령 금지 | "~하세요" | "~해보는 건 어때요?" |
| 공포 마케팅 금지 | "조심하세요, 나쁜 기운이" | "오늘은 마음을 천천히 다뤄주세요" |
| 호칭 | "회원님, 사용자님" | 닉네임으로 부르기 |

이 표가 그대로 LLM 시스템 프롬프트 + 템플릿 작성 가이드가 된다.

### 5.5 화면별 카피 기준점

- 랜딩: "사주, MBTI, 혈액형, 별자리… 흩어져 있던 '나'의 조각들을 하나로 이어드릴게요."
- 온보딩 생년월일시: "당신이 세상에 온 순간을 알려주세요. 그 순간의 하늘이 당신의 첫 이야기예요."
- 프로필 공개 로딩: "당신의 조각들을 잇는 중이에요…"
- 챗 첫 인사: "안녕하세요, {닉네임}님. 오늘 마음은 어떠세요?"
- 오류 화면: "잠시 길을 잃었어요. 금방 돌아올게요."

### 5.6 초기 포지셔닝

'분석'이 아닌 **'발견'과 '공감'** — "데이터를 통한 따뜻한 동행". 온보딩 카피도 "검사"가 아니라 "당신을 알아가는 시간"으로.

## 6. 사용자 참여 설계

### 6.1 데일리 리추얼 — 매일 돌아올 이유

- 자정에 '오늘의 가이드' 갱신. 구성: 오늘의 기운 한 줄 → 마음가짐 조언 → 오늘의 색/키워드 → 행운 포인트 (스크롤 30초 분량)
- 어제 가이드는 '지난 이야기'로 보관 — 놓쳐도 벌점 없음
- **스트릭 = '동행일'로 표현:** "옴니마인드와 함께한 지 14일째예요 🌱" — 리셋 표시 없이 누적 동행일 (압박 아닌 애착)
- 마일스톤: 7일(새싹)·30일(잎)·100일(나무)·365일(숲) — 프로필 배지 + 특별 카드 발급

### 6.2 공유 카드 — 무료 유입 엔진

- 생성 지점 3곳: ① 프로필 완성 직후("나의 조각 카드") ② 데일리 가이드("오늘의 나 카드") ③ 마일스톤 달성
- 규격: 인스타 스토리 9:16 + 정방형 1:1 두 종. 베이지 바탕 + 명조 제목 + 오행/별자리 심볼
- 내용 공식: **궁금증 유발형** — 답을 다 주지 않음. 예: "갑목(甲木)의 ENFJ, 사자자리 — 이 조합은 전체의 0.4%뿐이에요" + "나의 조합 보기 → 링크"
- 공유: 이미지 저장 / 카카오톡 / 링크 복사. 링크에 유입 추적 파라미터
- 배포 전략: SNS의 MBTI 콘텐츠 문법 활용 — "MBTI만으로는 반쪽짜리예요. 사주까지 이어야 진짜 나"

### 6.3 알림 — 다시 부르는 손길

- 1순위 카카오톡 채널, 2순위 웹 푸시(PWA)
- 아침 7~8시 1회. 예: "{닉네임}님, 오늘의 기운이 도착했어요 🍃 오늘은 '기다림'이 어울리는 날이에요" — 일부만 보여줌
- 절제: 하루 1회 초과 금지, 밤 발송 금지, 재촉 문구 금지

### 6.4 희소성 설계 — AI 챗 제한 UX

- 무료 하루 10회. 잔여 횟수를 "오늘 나눌 수 있는 이야기가 3번 남았어요"로 표현
- 소진 시: "오늘 이야기는 여기까지예요. 내일 아침, 새로운 기운과 함께 기다릴게요 🌙"
- 이 제한이 프리미엄 구독 1번 판매 포인트("마음 이야기 무제한")가 되도록 화면·문구에 자리를 심어둠

### 6.5 초대 루프 — 궁합/비교 (2차 로드맵, P7)

- "우리의 조합": 상대의 생년월일+MBTI 입력으로 관계 해석 (연인·친구·동료 모드)
- 상대가 직접 가입해 연결하면 양방향 심층 궁합 해제 → 초대 동기

### 6.6 성장 지표

| 지표 | 의미 | 도구 |
|------|------|------|
| 온보딩 완주율 | 첫 3분의 승부 | Vercel Analytics (무료) |
| 주간 재방문율 | 동반자 컨셉 검증 | 〃 |
| 공유 카드 생성/유입 수 | 바이럴 루프 작동 여부 | 유입 파라미터 |
| 챗 소진율 | 프리미엄 전환 잠재력 | DB 카운터 |

## 7. 릴리스 로드맵

매 단계가 그 자체로 배포 가능한 서비스가 되도록 구성. P2부터 실제 사용자를 받는다.

| 단계 | 내용 | 배포 시점 |
|------|------|-----------|
| **P0. 기반** | Next.js 스캐폴딩, 디자인 토큰, Supabase 연동, 카카오/구글 로그인 | 내부 확인용 |
| **P1. 계산 엔진** | 만세력(사주팔자·오행·십성), 별자리 산출, MBTI/혈액형 데이터 — 테스트 최우선 | 〃 |
| **P2. 온보딩+프로필** | 대화형 온보딩 4단계 → 템플릿 엔진 → "온전한 나" 프로필 | 🚀 소프트 런칭 |
| **P3. 데일리 가이드** | 홈 화면, 자정 갱신, 캐싱, 동행일 | 🚀 정식 출시 v1.0 |
| **P4. 확산 장치** | 공유 카드, 카카오톡 채널 알림, 광고(AdFit) 부착 | v1.1 |
| **P5. AI 챗** | Gemini 무료 티어 연동, 3단 폴백, 하루 10회 제한 UX | v1.2 |
| **P6. 의사결정 도우미** | '고민' 탭 — 고민 입력 → 프로필+운기 종합 조언 | v1.3 |
| **P7. 성장** | 궁합/초대 루프, 프리미엄 구독 전환 (유료 LLM 어댑터) | v2.0 |

## 8. 에러 처리

| 상황 | 대응 |
|------|------|
| LLM 쿼터 소진 / API 장애 | 템플릿 결과로 무중단 자동 대체 — 사용자에게 오류 노출 없음 |
| LLM 응답 이상 (형식 오류) | 1회 재시도 후 템플릿 폴백 |
| LLM 응답의 톤 위반 | 금지 표현 필터 미통과 시 템플릿 폴백 |
| 만세력 계산 경계 (자시 23시, 절기 경계일) | 경계 사례를 테스트로 고정, 출생 시간 미상 옵션 |
| Supabase 장애 | 캐시된 해석은 계속 표시, 신규 요청만 "잠시 길을 잃었어요" 화면 |

## 9. 테스트 전략

1. **계산 엔진 단위 테스트 (1순위):** 만세력은 정답이 존재 — 공개 만세력과 대조한 사례 수십 건을 테스트로 고정. 서비스 신뢰의 뿌리.
2. **템플릿 조합 테스트:** 어떤 유형 조합이 와도 문구 누락·중복 없이 생성되는지.
3. **E2E (Playwright):** 온보딩 → 프로필 공개 흐름 자동 검증.
4. **톤 검증:** 금지 표현 목록 기반 자동 검사 — 템플릿 DB와 LLM 출력 양쪽에 적용.
