## 2. 탭 구조

| 탭 | 경로 | 내용 |
|---|---|---|
| 홈 | `/` | 인사 + 6종 풀이 그리드(→ 사주팔자 탭) + 고객리뷰(실제 후기만, 0건 숨김 — P9 §5.2) + FAQ 발췌 3문항(기존 `FAQ_ITEMS` 재사용) + 전체 보기 |
| 오늘의운세 | `/today` | §3 참조 |
| 사주팔자 | `/saju` | §4 참조 |
| 보관함 | `/archive` | §5 참조 |

기존 `/me` `/mind` `/concern` `/match` 라우트는 **유지**(탭에서만 빠짐). 마음·고민은
로그인 사용자의 잠금 해제 화면(오늘의운세 전체 뷰·보관함)에서만 진입점이 보인다.

## 3. 오늘의운세 (`/today`, 달지기)

- **입력 팝업(바텀시트, 새창 아님):** 저장된 입력이 없으면 자동 표시. 생년월일·태어난 시
  (모름 허용 — 엔진 `timeUnknown`)·성별. 기존 `PickerInput`·`Choice` UI 재사용.
  입력은 **localStorage**(`om-today-birth`, JSON)에만 저장(비로그인 — 서버 저장 없음).
- **무료(비로그인):** 공통 일진만 — 기운·마음가짐·색·키워드·행운(기존 `assembleDaily`
  비개인화 경로). 개인 정보 기반 계산 결과는 무료 화면에 **없다**.
- **블러 티저:** 잠긴 카드 3장 — "내 일간으로 본 오늘"(십성 개인화) · "내 띠와 오늘"
  (지지 관계) · "AI가 다듬은 오늘의 이야기". **본문은 서버에서 아예 내려주지 않는다**
  (P9 §5.1) — 블러는 CSS 장식 자리표시자일 뿐, 실제 내용이 DOM에 없다.
  CTA: "로그인하고 모두 열어보기".
- **전체(로그인+프로필):** 기존 심화 일진(일간 십성·LLM 문단·하루기록·공유·근거 접이식)
  + **띠 관계 단락**(프로필 출생년 → `yearSign`·`branchRelation`·`relationLine` 재사용)
  + 마음 챗 진입 카드.
- **`/daily` 정리:** 년도 단독 흐름(직전 기능)은 이 탭으로 대체된다. `/daily` →
  `/today` 리다이렉트. `YearForm`·`DailySignSection`은 삭제(년도-단독 무료 개인화는
  결정 4와 상충). 엔진(`year-sign.ts`)·문구(`content/year-sign.ts`)는 전체 뷰에서 재사용.

## 4. 사주팔자 (`/saju`)

6종 풀이 카드 목록. 페르소나(7인 전담제, 2026-07-23 — `docs/persona-plan.md`):
서온=총운, 벼리=직업, 홍연=연애, 연리=궁합, 온새=결혼, 금오=재물.
접근은 전부 **로그인 무료**(등급 배지는 UI 비노출 — 숨긴 기능).

| 풀이 | id | 페르소나 | 접근(숨긴 메타) | 1단계 연결 |
|---|---|---|---|---|
| 총운 | `chongun` | 서온 | login | `/saju/chongun` |
| 직업 | `career` | 벼리 | login | `/saju/career` |
| 연애 | `love` | 홍연 | login | `/saju/love` |
| 재물 | `wealth` | 금오 | login | `/saju/wealth` |
| 궁합 | `match` | 연리 | login | `/saju/match-deep` |
| 결혼 | `marriage` | 온새 | login | `/saju/marriage` |

- 상품 카탈로그는 `src/lib/persona/products.ts`를 **v2로 교체**: `today` + 위 6종 = 7항목.
  홈 그리드·사주팔자 탭이 같은 카탈로그를 쓴다.
- 엿보기+블러(서버 비노출 잠금)·`readings` 캐싱·`readingAccess`는 **2단계**.
- 잠금 해제된 풀이 화면 안에 고민 상담 진입(2단계 이후).

## 5. 보관함 (`/archive`)

- **비로그인:** 로그인 게이트 — "기록을 남기고 다시 보려면 로그인" CTA.
- **로그인:** ① 오늘의운세 기록(기존 `interpretations` kind=daily 목록 — `/history`와
  같은 쿼리) ② 마음 대화(`/mind` 링크) ③ 고민 기록(`/concern` 링크)
  ④ 사주 풀이 기록(2단계 `readings`부터).
- `/history`는 유지하되 보관함이 표준 진입점.

## 6. 단계별 배포

| 단계 | 내용 | 상태 |
|---|---|---|
| **1** | 탭 4개 교체 + 홈 개편(그리드·FAQ 발췌·리뷰 자리) + `/today`(팝업·무료/블러·전체 뷰) + `/saju` 목록 + `/archive` + `/daily` 리다이렉트 | 화면 재배치 중심, 신규 상품 로직 없음 |
| **2** | 총운 전용 풀이 + `readings` 캐싱 + `readingAccess` + 엿보기/블러 실전화 | P9 2단계 흡수 |
| **3** | 크레딧 5종(직업·연애·재물·궁합 심층·결혼) | P9 3·4단계 흡수, 착수 시 상세화 |
| **4** | 후기 수집·노출(홈 고객리뷰 채움) + 공유 확장 | P9 5단계 흡수 |

## 7. 테스트 (1단계)

- 탭 4개: 라벨·경로·활성 상태 (BottomNav 테스트 갱신)
- 카탈로그 v2: 7항목·페르소나 매핑·접근 등급·1단계 href — 표와 전수 대조
- 오늘의운세: 무료 뷰에 개인화 본문 **부재**(블러는 자리표시자뿐) / 팝업 폼 렌더·localStorage 저장 / 톤 가드
- 홈: 그리드 6종 → `/saju`, FAQ 발췌 3문항, 리뷰 0건 → 섹션 숨김
- 보관함: 비로그인 게이트 문구(동기 컴포넌트로 분리해 테스트)
- 신규 카피 전부 톤 가드

## 8. 비목표 (1단계)

- 크레딧 차감·`readings`·`readingAccess`·실제 엿보기 본문(2단계)
- 후기 수집(4단계) — 홈 리뷰 섹션은 0건 숨김 상태로만 존재
- 비로그인 입력의 서버 저장(항상 localStorage만)
- 기존 `/me` `/match` 화면 자체의 개편(연결만 한다)
