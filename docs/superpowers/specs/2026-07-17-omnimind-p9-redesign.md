## 1. 개요

### 배경

경쟁 서비스 월문당(moonteller.net)은 검정 바탕·골드 포인트의 신비로운 무드 위에
페르소나 술사가 테마별 풀이를 파는 구조다. 엿보기 → 블러 → 결제 열람의 전환 동선이
명확하고, "30년 역술인 경력"이라는 **검증 불가능한 권위**로 신뢰를 만든다.

OmniMind는 그 구조적 강점(상품화된 풀이, 페르소나, 엿보기)은 가져오되,
신뢰 축은 **정반대로** 간다 — 지어낸 권위가 아니라 **검증 가능한 계산**이다.
이 저장소는 이미 절기를 USNO 공표 분점·지점과, 일진을 KASI 공표값 467건과 대조해
확정해뒀다. 이건 경쟁사가 흉내낼 수 없는 자산이며, P9의 `/sources`는 그걸 드러내는 장치다.

## 2. 상품 · 페르소나

### 2.1 라인업

| 상품 | 페르소나 | 접근 | 계산 기반 (전부 기존 엔진) | LLM |
|---|---|---|---|---|
| 오늘의 일진 | **달지기** | 누구나 무료 | `daily.ts` 일진 천간·십성 | 무료 티어 |
| 내 사주 심층 풀이 | **서온** | **로그인 무료** | `computeProfile` 전체 (일간·오행·십성·대운·12운성) | 무료 티어 |
| 인연 풀이 | **홍연** | 크레딧 | 대운·십성·12운성 | 프리미엄 |
| 궁합 심층 | **홍연** | 크레딧 | `computeMatch` / `computeDeepMatch` | 프리미엄 |
| 재물 풀이 | **금오** | 크레딧 | 십성(재성)·대운 | 프리미엄 |

**과금 원칙:** 무료 풀이는 **무료 티어 모델만** 쓴다. 유료 모델은 크레딧 소비 시에만.
이로써 `월 고정비 0원` 원칙이 유지된다(CLAUDE.md).

### 2.2 3단계 티어 (기존 P8 구조 그대로)

- **비로그인** — 오늘의 일진 + 온보딩 프로필 미리보기. 심층 풀이는 엿보기(블러)까지만,
  CTA는 "로그인하고 무료로 열람".
- **로그인** — 내 사주 심층 풀이 **전체 무료** + 보관함 저장 + 기존 마음·고민 하루 1회.
- **크레딧** — 인연·궁합·재물 풀이.

> **설계상의 이득:** 엿보기·블러 컴포넌트가 두 곳에서 재사용된다 —
> 비로그인에겐 "로그인하면 무료"의 훅으로, 로그인 사용자에겐 크레딧 상품의 판매 장치로.
> **잠금 사유(`lockReason`)만 다르고 컴포넌트는 하나다.** 이게 설계를 단순하게 만든다.

### 2.3 페르소나

페르소나는 **표현 계층 전용**이다. 계산에 일절 관여하지 않는다(계산과 해석의 분리 원칙).
구체적으로 페르소나가 정하는 것은 두 가지뿐이다:

1. UI에 노출되는 고정 대사 (홈 카드 멘트, 상품 페이지 인사)
2. LLM 프롬프트의 **말투 지시문** — 내용이 아니라 어조만

| 페르소나 | 정체 | 말투 | 담당 |
|---|---|---|---|
| **달지기** | 밤마다 등불을 켜는 문지기 | 부드러운 존댓말 | 오늘의 일진 |
| **서온** | 서고를 지키는 이 | 차분한 존댓말 | 내 사주 심층 |
| **홍연** | 붉은 실을 잇는 이 | 다정한 반말 | 인연 · 궁합 |
| **금오** | 금까마귀 | 호쾌한 하오체 | 재물 |

**문체 제약:** 페르소나가 생겨도 §5.4 '지적인 따뜻함'은 유지된다.
월문당식 공포·조급 유발("~하지 않으면 놓친다", "지금 확인하지 않으면")은 **쓰지 않는다**.
페르소나는 캐릭터일 뿐 압박 장치가 아니다.

**데이터 구조:** `src/lib/persona/personas.ts` — 순수 상수. 엔진과 무관.

```ts
export interface Persona {
  id: PersonaId;          // "dalzigi" | "seoon" | "hongyeon" | "geumo"
  name: string;           // "서온"
  title: string;          // "서고를 지키는 이"
  homeLine: string;       // 홈 카드 5초 멘트
  greeting: string;       // 상품 페이지 인사
  toneInstruction: string; // LLM 말투 지시문 (내용 아님)
}
```

## 3. 해석 축 위계 (모든 풀이 공통 규칙)

### 3.1 위계

1. **주축 — 팔자**: 일간·일진·오행·십성·대운·12운성.
   섹션 구조와 **결론**을 결정한다.
2. **보조축 — MBTI·혈액형**: 주축이 내린 결론이 *"그래서 당신에겐 어떻게 드러나는가"* 를
   설명한다. **결론을 뒤집지 않는다.**
3. **최경량 — 별자리**: 인사·무드.

### 3.2 구속 규칙

- 보조지표 **단독으로 섹션의 결론을 만들지 않는다.**
- **문장 순서가 곧 위계다** — 팔자 문장이 먼저, 보조 문장이 뒤에서 수식한다.
- 보조지표가 없어도(비로그인·구버전 프로필) 풀이는 **완전히 동작한다.** 폴백 필수.

### 3.3 현재 코드 실사 결과

| 대상 | 현재 상태 | P9 작업 |
|---|---|---|
| `assembleProfile` | ✅ **이미 준수** — 일간→오행→십성으로 팔자가 뼈대, MBTI·혈액형은 '겉과 속' 한 섹션에서 보조 | 심층 풀이로 확장 시 위계 유지 |
| `synthesisText` | ✅ **이미 준수** — "십성이 주어, MBTI가 수식" ("표현하고 싶은 사주의 기운이 E의 결을 타고…") | **이 패턴을 다른 풀이에 재사용** |
| `assembleDaily` | ❌ **보조축 없음** — 일진 천간 무드 + 십성 개인화까지만. 혈액형·MBTI 미사용 | **보조 한 줄 추가** |

### 3.4 일진 보조축 추가

`assembleDaily`의 기존 `personal`(일진 십성 한 줄) **뒤에** 보조 한 줄을 덧붙인다.
`DailyGuide`에 `secondary: string | null` 필드 추가.

- **교차 축**: 오늘의 십성(주축) × MBTI E/I × 혈액형(보조)
- **패턴**: `synthesisText`와 동일 — 사주가 주어, 보조가 수식
- **폴백**: 프로필 없음 / MBTI·혈액형 없음 → `secondary = null`, 일진은 지금처럼 완전 동작

예시(상관의 날 + ENFJ + A형):
> `personal`: "오늘은 표현이 반짝이며 튀어나오는 상관의 기운이에요…" (주축)
> `secondary`: "이 기운이 ENFJ의 결과 만나 유난히 잘 흐르는 날이에요. 다만 A형 특유의
> 조심스러움이 한 번 더 다듬게 하니, 그 사이의 균형만 잡아주면 돼요." (보조)

> **왜 새 발명이 아닌가:** `synthesisText`가 이미 쓰는 검증된 교차 패턴을 넓히는 것이다.
> 새 계산은 없다 — 이미 `DailyContext.tenGod`과 `ProfileContext.mbti/blood`가 있다.

## 4. 디자인 시스템 — "밤의 따뜻함"

### 4.1 방향

월문당의 신비로운 밤 무드를 가져오되, **공격적 톤은 가져오지 않는다.**
검정(#0a0a0a)이 아니라 **깊은 밤 네이비** — 차갑지 않고 잠기는 느낌.
골드는 화려한 금(#ffc83d)이 아니라 **달빛 골드**(#f0c96a) — 은은한 온기.

기존 브랜드 가이드(§5.2 "차가운 블루톤 지양")와의 관계: 밤 네이비는 채도를 낮추고
따뜻한 쪽으로 기울인 값(#0e1626)이라 '차가운 블루'가 아니다. 코랄은 그대로 살아남는다.

### 4.2 토큰

컴포넌트는 이미 토큰만 참조하므로 **`globals.css` 토큰 교체가 작업의 중심**이다.

| 토큰 | 다크(기본 무드) | 라이트('새벽') |
|---|---|---|
| `--warm-base` | `#0e1626` 밤 네이비 | `#f5efe6` (유지) |
| `--warm-surface` | `#1a2740` | `#fdfbf7` (유지) |
| `--text-main` | `#f0ebe2` 웜 아이보리 | `#3e3a36` (유지) |
| `--text-soft` | `#a8b0c0` | `#8a8178` (유지) |
| `--moon-gold` | `#f0c96a` **(신규)** | `#c9a24a` |
| `--accent-coral` | `#e8927c` (유지) | `#e8927c` (유지) |
| `--selected` / `--on-selected` | `#f0c96a` / `#0e1626` | `#2d5a4a` / `#ffffff` |

**기존 `selected` 토큰 재사용:** 오늘 커밋한 `--selected`/`--on-selected`가 그대로 살아있다.
다크에서 코랄 → **달빛 골드**로 값만 바뀐다. 컴포넌트 수정 0.

**모티프:** 달·별은 **CSS만으로**(radial-gradient + 작은 점 애니메이션). 이미지 없음.
`prefers-reduced-motion` 필수 대응 — 기존 모션 토큰(`--ease-out`) 재사용.

### 4.3 홈 CSS 모션 카드

> **실제 영상을 기각한 이유(기록):** 5초 영상 5개 = 홈 1뷰당 5~10MB.
> Vercel 무료 100GB/월 → **월 1~2만 홈뷰에서 소진**. 유료 플랜 필요 = 고정비 0원 원칙 위배.
> 또한 모바일에서 5개 동시 자동재생은 iOS 정책·발열·데이터 소모로 비현실적이며,
> 실제 숏폼 앱도 뷰포트 1개만 재생한다. 카피를 영상에 굽으면 수정 시 재생성해야 하고
> 스크린리더·SEO에도 잡히지 않는다.

**대안: 5초 CSS 루프 카드** — 카드당 ~50KB(일러스트 webp 1장), 모션은 CSS(0 바이트 추가).

5초 루프 구성:
| 구간 | 모션 |
|---|---|
| 0~5s (배경) | 달빛 스윕(gradient 이동) · 별 반짝임(opacity) |
| 0~5s (캐릭터) | 살짝 부유(translateY ±3px) |
| 0.3~1.2s | 페르소나 멘트 페이드인 |
| 2.5s~ | CTA 은은한 펄스 |

- 멘트는 **진짜 텍스트** → 수정 자유 · 스크린리더 읽힘 · SEO 반영
- `prefers-reduced-motion: reduce` → 모션 전부 정지, 정적 카드로 완전 동작
- 일러스트가 아직 없으면 **CSS 심볼**(달·실·까마귀 등 도형)로 폴백 — 배포를 막지 않는다

### 4.4 홈 레이아웃

```
┌────────────────────────┐
│ 인사 + 오늘 밤 한 줄      │
│ ┌────────────────────┐ │
│ │ 오늘의 일진 (무료)    │ │  ← 달지기 · 매일 갱신 · 무료 훅
│ └────────────────────┘ │
│ ┌────────────────────┐ │
│ │ 내 사주 심층 (서온)   │ │  ← 로그인하면 무료
│ ├────────────────────┤ │
│ │ 궁합 심층  (홍연)     │ │  ← 크레딧
│ ├────────────────────┤ │
│ │ 인연 풀이  (홍연)     │ │  ← 크레딧
│ ├────────────────────┤ │
│ │ 재물 풀이  (금오)     │ │  ← 크레딧
│ └────────────────────┘ │
│ 마음 챗 · 고민 진입       │
│ 후기 하이라이트 (있을 때만) │
│ 푸터                    │
└────────────────────────┘
```

하단 탭바는 **기존 유지**. 홈 화면 내용만 개편한다.

## 5. 판매 장치

### 5.1 엿보기 + 블러 (전환의 핵심)

상품 페이지 구조: **페르소나 인사 → 엿보기 3~4개 → CTA**

엿보기 1개 = `{ title, teaser(공개), body(잠김), bullets(잠김) }`
- 잠김 상태: `body`를 블러 처리 + 하단 그라데이션 + bullets는 자물쇠 아이콘
- **블러는 시각 효과일 뿐이므로, 잠긴 본문은 서버에서 아예 내려주지 않는다.**
  DOM에 넣고 CSS로 가리면 개발자 도구로 다 읽힌다. 잠김 시 `body: null`.

`lockReason` 3종으로 CTA가 갈린다:
| lockReason | CTA | 대상 |
|---|---|---|
| `null` | 없음(전체 공개) | 열람 권한 있음 |
| `"login"` | "로그인하고 무료로 열람" | 비로그인 × 심층 풀이 |
| `"credit"` | "크레딧 1회로 열기" | 로그인 × 유료 풀이 |

**공포 마케팅 금지:** 카운트다운·"지금 아니면 사라져요"·가짜 할인은 쓰지 않는다(§2.3 문체 제약).

### 5.2 후기

- 풀이 열람 **후** 별점(1~5) + 한 줄 후기 수집
- **실제 후기만** 노출. 개수가 적으면 섹션 자체를 숨긴다 — 빈 상태를 꾸미지 않는다
- 가짜 후기·부풀린 카운트("200% 실제 후기" 같은 문구)는 **만들지 않는다**
- 노출: 상품 페이지 하단 + (충분히 쌓이면) 홈

### 5.3 공유 훅

기존 초대 링크 + `api/card`를 **풀이 결과 공유 카드**로 확장.
결과 요약 카드 이미지 + 유입 링크. 새 인프라 없음.

## 6. 데이터 모델

### 6.1 신규 테이블

```sql
-- 풀이 캐싱 — "같은 해석을 두 번 생성하지 않는다"(CLAUDE.md 캐싱 원칙)
create table readings (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id) on delete cascade,
  product text not null,              -- "profile_deep" | "match_deep" | "fate" | "wealth"
  input_hash text not null,           -- 입력(프로필/상대) 해시 — 바뀌면 재생성
  context_version int not null,       -- PROFILE_CONTEXT_VERSION — 낮으면 재계산 대상
  sections jsonb not null,
  created_at timestamptz not null default now(),
  unique (user_id, product, input_hash)
);

create table reading_reviews (
  id uuid primary key default gen_random_uuid(),
  reading_id uuid not null references readings(id) on delete cascade,
  user_id uuid not null references auth.users(id) on delete cascade,
  rating int not null check (rating between 1 and 5),
  comment text check (char_length(comment) <= 200),
  created_at timestamptz not null default now(),
  unique (reading_id)                 -- 풀이 1건당 후기 1개
);

create table inquiries (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users(id) on delete set null,  -- 비로그인 허용
  email text not null,
  subject text not null,
  body text not null,
  created_at timestamptz not null default now()
);
```

RLS: 세 테이블 모두 본인 행만 select/insert. `reading_reviews`의 공개 노출은
집계 뷰(별점 평균·개수·코멘트)로만 — 작성자 식별 정보는 내리지 않는다.

### 6.2 캐싱 규칙 (비용 통제의 핵심)

- `input_hash`가 같고 `context_version`이 현재 값 이상이면 **저장된 것을 그대로 반환.**
  LLM 호출 0. 재열람은 언제나 공짜.
- 프로필이 바뀌거나 엔진 의미가 바뀌면(`PROFILE_CONTEXT_VERSION` 상승) 재생성 대상.
- **크레딧은 새 생성 시에만 차감.** 재열람에 재차감하지 않는다.

### 6.3 접근 규칙

`src/lib/consult/quota.ts` **하나에** 상품별 규칙을 모은다(CLAUDE.md 원칙 유지).
새 모듈을 만들지 않는다.


## 7. 출처 (`/sources`)

### 7.1 왜 만드는가

월문당의 신뢰 축은 "30년 역술인 경력" — 검증할 수 없다.
OmniMind의 신뢰 축은 **검증 가능한 계산**이다. 이 페이지는 그걸 드러낸다.
**이건 마케팅 카피가 아니라 사실의 공개이며, 사실이기 때문에 강하다.**

### 7.2 내용

**계산 (검증 가능 — 자신 있게 공개)**
- 절기: astronomy-engine 태양 겉보기 황경, 1900~2100 초 단위 저장.
  **USNO 공표 분점·지점과 대조 검증**(`solar-terms.usno.test.ts`, CI 상시)
- 일진: 2000-01-07 갑자일 앵커 JDN 산술.
  **KASI '음양력 정보' 공표 일진 467건(1900~2050)과 대조 확정**(`iljin.test.ts`)
- 시간 보정: 서머타임(−1h), 표준시 UTC+8:30 시대(+30m), 야자시 23시 경계

**해석 (정직하게 고지)**
- 전통 명리 이론 체계(일간·오행·십성·대운·12운성)에 기반하며, **문장은 옴니마인드가 쓴다.**
- **고전 문헌을 문장별 전거로 표기하지 않는다.** 그건 사실이 아니고, 지어낸 인용은
  이 페이지의 존재 이유를 무너뜨린다.

**AI 고지**
- 계산은 100% 규칙 기반이며 **AI가 관여하지 않는다.**
  AI는 이미 계산된 결과를 당신에게 맞는 문장으로 다듬는 일만 한다.

**면책**: 참고용이며 의료·법률·투자 판단의 근거가 아니다.

### 7.3 배치

- `/sources` 전체 페이지
- 각 풀이 하단 "이 풀이의 근거" 접이식 — 핵심 3줄 + 전체 보기 링크

## 10. 단계별 배포

각 단계 종료 시점에 **서비스가 온전히 동작**하며, 단계마다 배포한다.

| 단계 | 내용 | 배포 후 상태 |
|---|---|---|
| **1** | 디자인 토큰 전환(밤의 따뜻함) + 홈 개편(CSS 모션 카드) + 푸터 + `/sources` `/faq` `/contact` `/terms` `/privacy` | 새 옷을 입은 기존 서비스. 상품은 아직 기존 것 |
| **2** | **내 사주 심층 풀이(서온)** + 엿보기·블러 + `readings` 캐싱 + `readingAccess` | 무료 상품으로 열람 파이프라인 검증 — **결제가 안 끼어 있어 안전** |
| **3** | **궁합 심층(홍연)** — 2단계 파이프라인에 크레딧 차감만 얹음 | 첫 유료 풀이 |
| **4** | 인연(홍연) · 재물(금오) + 일진 보조축(§3.4) | 라인업 완성 |
| **5** | 후기 수집·노출 + 공유 카드 확장 | 성장 루프 |

> **2단계를 무료 상품으로 두는 것이 핵심이다.** 열람·캐싱·잠금 파이프라인을 결제 없이
> 먼저 검증하고, 크레딧은 검증된 구조 위에 얹는다. 3단계는 그래서 작아진다.
