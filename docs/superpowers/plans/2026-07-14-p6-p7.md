# P6 의사결정 도우미 + P7 성장 구현 플랜 (2026-07-14)

## P6. '고민' 탭 — ✅ 코드 완료

**흐름:** 카테고리 선택 + 자유 서술 → `submitConcern()` → 템플릿 조언(항상) + LLM 개인화(가능할 때) → `interpretations(kind=advice)` 보관 → '지난 고민들' 다시 보기.

| 모듈 | 역할 |
|------|------|
| `src/lib/interpret/content/concern.ts` | 카테고리 4종(일·커리어/관계/선택·결정/마음) × 오행 관계 5종 조언 조립 + LLM 프롬프트. 전 조합 톤 검증 테스트 |
| `src/lib/concern/actions.ts` | 서버 액션 — 하루 3회 제한(advice 행 created_at, KST 자정 기준), 3단 폴백 재사용(`respond()` 주입) |
| `src/lib/concern/constants.ts` | `CONCERN_DAILY_LIMIT=3`, 서술 500자 |
| `src/components/concern/ConcernRoom.tsx` | 카테고리 칩 + 입력 + 결과 카드 + 지난 고민(details) |
| `src/app/(tabs)/concern/page.tsx` | 프로필 없으면 온보딩 CTA, 있으면 ConcernRoom |

**설계 결정:** `interpretations`의 `unique(user_id,kind,target_date)`는 advice에서 `target_date=null`로 우회(Postgres에서 null은 중복 허용) — 마이그레이션 없이 하루 여러 건 보관. LLM 실패 시 개인화 문단만 조용히 생략(무중단 원칙 §8).

## P7. 성장 — 🟡 착수

### P7-1 궁합 "우리의 조합" ✅

**흐름:** 내 프로필(일간 오행·별자리·MBTI) × 상대(생년월일 + MBTI 선택) → 순수 계산 → 해석 카드 → 초대 링크.

| 모듈 | 역할 |
|------|------|
| `src/lib/engine/match.ts` | TDD. `partnerFromBirth`(정오 기준 일주→오행, 별자리), `relateElement` 재사용(동행/채움/발산/결실/단련), 별자리 4원소 조화(닮음/어울림/다름), `mbtiSynergy` 0~5, 모드 가중 점수 0~100(결정적) |
| `src/lib/interpret/content/match.ts` | "우리의 온도 N°" + 관계 해석 5종 + 원소 조화 3종 + 성향 3단 + 모드 마무리. 서열화·공포 금지, 전 조합 톤 통과 |
| `src/components/match/MatchForm.tsx` | 모드 칩(연인/친구/동료) + 입력 → 클라이언트 계산(엔진 순수함수) + 초대 링크 복사(`?ref=match&via=<slug>`) |
| `src/app/(tabs)/match/page.tsx` | 진입: `/me`의 "우리의 조합 보기" 버튼 |
| `src/lib/metrics/names.ts` | `match_compute`, `match_copy_link` 이벤트 추가 |

### P7-2 양방향 심층 궁합 — 미착수 (다음 단계)

상대가 초대 링크로 가입·연결하면 두 프로필(사주 전체)로 심층 해석 해제. `connections` 테이블 + 수락 흐름 필요.
